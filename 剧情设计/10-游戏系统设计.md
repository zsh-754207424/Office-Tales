# 游戏系统设计文档

## 1. 概述

### 1.1 游戏类型
- **类型**: 职场题材视觉小说（Visual Novel）
- **视角**: 第一人称叙事
- **预估时长**: 约5小时（单次通关），24结局全解锁约15小时
- **目标平台**: PC / Mac / 移动端

### 1.2 核心机制
- **选择分支**: 玩家通过4选项选择推进剧情
- **数值系统**: 好感/信任/敌意影响剧情走向和结局
- **多结局**: 24种结局，基于派系立场、信任度、关键选择三轴

---

## 2. 数值系统

### 2.1 角色数值

| 数值名称 | 范围 | 初始值 | 核心作用 |
|---------|------|--------|---------|
| 李总监信任 | 0-100 | 50 | 决定晋升路线和A/B类结局 |
| 王浩敌意 | 0-100 | 40 | 影响竞争强度和解锁特殊对话 |
| 张姐好感 | 0-100 | 60 | 影响情报获取和帮助意愿 |
| 陈叔好感 | 0-100 | 30 | 影响秘密揭示程度 |
| 小刘HR好感 | 0-100 | 40 | 影响内部消息获取 |
| 小林好感 | 0-100 | 50 | 影响小林结局和B故事收束 |

### 2.2 数值变化规则
- 选择分支每次增减范围：±5 ~ ±20
- 主要转折点（第2章抢功事件、第3章站队等）影响值更大
- 部分选择有隐藏数值变化（不直接显示）
- 累积效果导致结局分支

### 2.3 隐藏数值

| 数值名称 | 范围 | 说明 |
|---------|------|------|
| 派系立场 | 李/中/王 | 关键选择的累积结果 |
| 能力评价 | 低/中/高 | 影响晋升可能 |
| 孤立度 | 0-100 | 影响是否被边缘化 |
| 道德值 | 0-100 | 影响特定结局解锁 |
| 小林命运 | 好/中/差 | 基于玩家的帮助程度 |
| 王浩把柄 | 有/无 | 是否掌握王浩的把柄 |

## 3. 选择分支系统

### 3.1 选择结构
每个选择节点包含：
1. **场景上下文** — 玩家做出选择前的情景描述
2. **选项文本**（通常4个）— 玩家看到的选择
3. **选项结果** — 选择后的角色反应 + 内心独白
4. **数值影响** — 具体的数值增减
5. **后果反馈** — 后续章节中的连锁反应

### 3.2 选择类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 价值选择 | 反映价值观 | 站李总监vs站王副总 |
| 策略选择 | 决定行动策略 | 当面质问vs收集证据 |
| 关系选择 | 影响人际关系 | 帮助小林vs冷淡回应 |
| 道德选择 | 涉及道德判断 | 接受暗箱操作vs拒绝 |

### 3.3 三轴结局判定

结局由以下三个维度的累积值决定：

```
X轴：派系立场
├─ 李总监派系（X=0~1）
├─ 中立（X=2~3）
└─ 王副总派系（X=4~5）

Y轴：李总监信任度
├─ 高（≥60）
├─ 中（40-59）
└─ 低（<40）

Z轴：核心选择
├─ 关键时刻正确
├─ 中规中矩
└─ 关键时刻错误
```

## 4. 存档与进度系统

### 4.1 自动存档
- 每章开头自动存档
- 每个关键选择节点前自动存档
- 结局触发时自动存档

### 4.2 手动存档
- 玩家可在任意非选择界面手动存档
- 存档数量上限：20个

### 4.3 进度追踪
- 结局解锁进度：24种结局的解锁状态
- 选择回顾：已做选择的完整记录

---

## 5. 结局解锁系统

### 5.1 判定流程
1. 第4章结束时，系统计算三轴累积值
2. 根据矩阵对照表确定结局编号
3. 第5章渲染对应结局内容

### 5.2 结局矩阵

|          | 派系正确 | 中立 | 派系错误 |
|----------|----------|------|----------|
| **信任高** | A1-1,A1-2,A1-3 | A2-1,A2-2,A2-3 | C1-1,C1-2,C1-3 |
| **信任中** | B1-1,B1-2 | B2-1,B2-2 | C2-1,C2-2,C2-3 |
| **信任低** | B3-1,B3-2 | C3-1,C3-2 | C3-3 |
